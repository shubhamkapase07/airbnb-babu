Scaling notes. Frontend: static/ISR listing pages cached at the edge globally; image CDN with on-the-fly resize; RSC streaming to cut TTFB. Backend: stateless services autoscaled per traffic; read replicas + region-sharded Postgres; write-through Redis for hot listings. Search: Elasticsearch cluster fed asynchronously from the event bus so search never blocks writes. Storage: photos in S3 behind CDN, DB holds only URLs/metadata. Reliability: idempotent booking transactions, distributed locks on availability, event-sourced audit trail, multi-AZ + multi-region failover.
